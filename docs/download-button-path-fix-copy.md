# 下载按钮显示异常修复方案（副本）

## 问题原因

- 前端下载按钮的显示逻辑依赖路径正则匹配（通常匹配 `/download/...`）。
- 后端 `PDFGenerationTool` 返回的是磁盘绝对路径（如 `F:/.../tmp/pdf/xxx.pdf`），不满足前端正则。
- 结果是前端无法识别为“可下载地址”，下载按钮不显示。

## 修改目标

- 后端统一返回可被前端直接识别的 URL 格式：`/download/pdf/xxx.pdf`。
- 同时保留本地路径信息，避免影响已有日志或排查流程。

## 实施内容

- 修改文件：`src/main/java/com/ui/ailvyou/tools/PDFGenerationTool.java`
- 新增逻辑：
  - 将生成后的本地文件路径转换为对外下载 URL（`/download/**`）
  - 对 URL 路径分段做编码，兼容中文文件名和空格
  - 返回文案改为同时包含“下载地址 + 本地路径”

## 预期效果

- 前端正则能稳定匹配 `/download/...` 格式。
- 下载按钮正常显示。
- 文件名为中文/含空格时，下载链接仍可访问。

## 注意事项

- 该方案依赖 `WebConfig` 的资源映射：`/download/** -> file:./tmp/`。
- 若后续前端正则调整，建议以“匹配 URL”而非“匹配磁盘路径”作为唯一规则。
